# Device profile: Govee H5074 / H5075 (BLE temp+humidity)

Field-confirmed during a real parser-diff investigation (radio capture vs Home Assistant). Both are
passive BLE broadcasters; **the temp/humidity payload rides in the SCAN RESPONSE**, so you must use
**active scanning** to see it (passive yields only the name PDU).

- **Company ID:** `0xEC88` (shared by BOTH models — you cannot tell them apart from the company id).
- Both payloads **lead with `0x00`**. Decode by **known-model-per-MAC**, not by guessing from bytes.

## Manufacturer-data payload (bytes after the `0xEC88` company id)

### H5074 — 7 bytes
```
00 | temp(int16 LE)/100 | hum(int16 LE)/100 | battery | <1 trailing>
[0]   [1:3]                [3:5]               [5]        [6]
```
`temp_c = int16_le(d[1:3]) / 100`, `humidity = int16_le(d[3:5]) / 100`, `battery = d[5]`.

### H5075 — 3-byte packed
```
00 | packed(3-byte BE) | battery
[0]   [1:4]              [4]
```
`packed = int.from_bytes(d[1:4], "big")`; top bit (`0x800000`) is the temperature sign, mask it off.
`temp_c = ±packed/10000`, `humidity = (packed % 1000) / 10`, `battery = d[4]`.

## Decoder

`scripts/decode_ble_log.py` implements both (with ANSI-strip + SUSPECT flagging) and is the reference
for adding more models. Confirm any new model's shape with a one-variable diff against the device
screen / vendor app before trusting it.

## Known failure mode (the reason this profile exists)

An H5074 reporting humidity correctly *most* of the time but **spiking wrong specifically when
humidity rises** (e.g. a shower) — while a co-located **H5075 stays correct** — points NOT at a
constant offset/scale parser bug (a clean baseline rules that out) but at an **event-driven**
problem: either a pipeline/parse glitch on the rising edge, or an **over-the-air MAC collision**
(another broadcaster transmitting on the H5074's MAC). The witness radio's **SUSPECT** flags
(a non-`0xEC88` company id — e.g. Apple `0x004C` — or a foreign name on the Govee MAC) adjudicate
the collision hypothesis; the H5075 is the control. This is exactly what the parser-diff loop is for.
